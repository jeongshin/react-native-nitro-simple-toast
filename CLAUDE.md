# react-native-nitro-simple-toast

## Motivation

A native toast library for React Native that displays toasts **above modals and all other views** at the native level. Inspired by [burnt](https://github.com/nandorojo/burnt), but redesigned using [Nitro Modules](https://nitro.margelo.com/) to eliminate the Expo dependency.

## Architecture

- **Framework**: Nitro Modules (HybridObject pattern)
- **iOS**: [SPIndicator](https://github.com/ivanvorobei/SPIndicator) - native floating indicator
- **Android**: `android.widget.Toast` (Android SDK) with haptic feedback via `Vibrator`
- **No web support** - iOS and Android only
- **No alert, no custom icons, no SF Symbols** - just toast

## Project Structure

```
src/
  NitroSimpleToast.nitro.ts   # Nitro spec (types + HybridObject interface)
  index.tsx                    # Public API: toast() function
ios/
  NitroSimpleToast.swift       # iOS implementation (SPIndicator)
android/
  .../NitroSimpleToast.kt      # Android implementation (android.widget.Toast)
  .../NitroSimpleToastPackage.kt # React Native package registration
nitrogen/generated/            # Auto-generated by nitrogen CLI (do not edit)
example/                       # Example app for testing
```

## Key Commands

- `npx nitrogen` - Regenerate native specs after changing `.nitro.ts` files
- `yarn typecheck` - TypeScript type checking
- `cd example/ios && pod install` - Install iOS dependencies

## Conventions

- Enums use Nitro string union types: `type ToastPreset = 'done' | 'error' | 'none'`
- Interfaces (non-HybridObject) become native structs
- `show()` is synchronous (fire-and-forget, <50ms) - no Promise needed
- iOS-only options (`from`, `shouldDismissByDrag`) are silently ignored on Android
